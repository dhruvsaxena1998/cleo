## 1. Hook Diagnosis

### 1.1 `cleo events <session-id>`

New top-level command. Reads `~/.config/cleo/events/<session-id>.jsonl` (or the matching archive under `events/archive/` if the session was pruned).

**Command shape:**

```
cleo events <session-id> [-f|--follow] [--type <kind>] [--since <duration>] [-n|--limit <N>] [--json]
```

**Flags:**

- *(default)* ‚Äî print all events sorted by timestamp ascending. Tab-separated columns: `TIME TYPE STATE MESSAGE`. Lipgloss-styled when stdout is a TTY, plain when piped.
- `-f`, `--follow` ‚Äî tail the file. Reopen on inode change so a `cleo prune` archive doesn't kill the tail. Poll-based, 500 ms cadence.
- `--type <kind>` ‚Äî filter to one event kind (e.g. `notification`, `pre_tool_use`).
- `--since <duration>` ‚Äî only events newer than `now - duration`. Accepts Go duration strings.
- `-n <N>`, `--limit <N>` ‚Äî show only the most recent N events. Mutually exclusive with `--follow`.
- `--json` ‚Äî emit the raw JSONL lines unchanged for `jq` pipelines.

**Session resolution:**

1. Exact match against active sessions in `state.json`.
2. Exact match against archived event files under `events/archive/`.
3. Substring match against active+archived. Error if multiple match.

**File location.**

```
internal/cli/events.go
internal/cli/events_test.go
```

Register in `internal/cli/root.go`.

**Behavior on missing file.** Active session with empty events ‚Üí print `(no events yet)` to stderr, exit 0. Unknown session ‚Üí exit 1 with `unknown session: <id>`.

### 1.2 `cleo doctor` Upgrade

Keep the four existing checks (Claude hook entries, Codex feature flag, Codex hook entries, recent activity). Add three sections.

**Inline last 3 hook traces (per agent).**

After each agent's hook check, append a sub-block formatted:

```
  Last hook traces:
    14:02:11  notification     cleo-myapp-claude-fix-auth   env_present
    14:01:03  post_tool_use    cleo-myapp-claude-fix-auth   env_present
    13:58:47  session_start    cleo-myapp-claude-fix-auth   env_present
```

Pulled from `hook-trace.log`. Up to 3 most recent per agent. The right-most column is `fallback_reason` (item 1.3). Surfaces "cleo is hearing your hooks" without grepping.

**Attribution-failure summary.**

Scan `hook-trace.log` for entries with `fallback_reason ‚àà {env_unknown_session, no_match}` since `now - 24h`. Print:

```
Attribution failures (last 24h): 4
  Last 3:
    13:55:02  /Users/me/Dev/foo  notification         no_match
    13:54:51  /Users/me/Dev/foo  pre_tool_use         no_match
    13:51:18  /Users/me/Dev/bar  notification         env_unknown_session
```

Empty section omitted when count is 0.

**Hook-config diff.**

Compute the entries `cleo init` would install for each agent vs. what's on disk. Print:

```
Claude hooks (~/.claude/settings.json):
  = SessionStart       cleo hook claude
  = UserPromptSubmit   cleo hook claude
  + PreToolUse         cleo hook claude    (would install)
  - Stop               (foreign entry: /usr/local/bin/other)
```

Line markers: `=` matches expected, `+` cleo would add (run `cleo init`), `-` cleo's expected entry is missing or replaced. Empty diffs print `Claude hooks: in sync ‚úì`.

**`--quiet` flag.**

Suppresses passing checks; only prints failures, the attribution-failure summary if non-empty, and the hook-config diff if non-empty. Empty output and exit 0 on a fully-healthy install. Cron- and CI-friendly.

**File changes.**

```
internal/cli/doctor.go        (extend existing)
internal/cli/doctor_test.go   (add fixture-based diff/summary tests)
```

### 1.3 Attribution Traces ‚Äî `fallback_reason`

Pure logging change in `internal/hooks/handler.go`. The current resolver picks a session via:

1. `CLEO_SESSION_ID` env var if set and matches.
2. Hook payload `cwd` ‚Üí most-recently-active session for that project+agent.

Add a `fallback_reason` field to each trace log entry, with one of:

- `env_present` ‚Äî env var set and matched.
- `env_missing` ‚Äî env var unset; fell through to cwd lookup.
- `env_unknown_session` ‚Äî env var set but no state entry. Fell through.
- `multi_match_first` ‚Äî multiple cwd-matching candidates; picked by most recent `LastEventAt`.
- `no_match` ‚Äî no resolution. Today this is silent; v0.2 also writes a line to `hook-errors.log`.

No behavior change. The doctor upgrade (1.2) consumes this field.

**Test additions.** One unit test per reason in `internal/hooks/handler_test.go`.

## 2. TUI Polish

### 2.1 Pane Preview Reliability Rewrite

The most user-visible v0.2 change. Six concrete bugs identified by reading `internal/tui/{poll,update,handle_key,main_pane}.go` and `internal/tmux/tmux.go`.

**Bug A ‚Äî `pane_preview_lines` is silently ignored.**

`internal/tmux/tmux.go:99` declares `CapturePane(name string, _ int) (string, error)` and discards the `lines` parameter. The shell-out is `tmux capture-pane -p -t <name>:.` with no `-S` flag ‚Äî visible-pane only, no scrollback. The user-facing config `pane_preview_lines` (default 30) does literally nothing.

**Fix:** consume the parameter:

```go
func (c *Client) CapturePane(name string, lines int) (string, error) {
    if lines <= 0 {
        lines = 30
    }
    out, err := c.cmd("capture-pane", "-p", "-S", fmt.Sprintf("-%d", lines), "-t", name+":.").Output()
    return string(out), err
}
```

**Bug B ‚Äî capture loop dies silently after fast navigation.**

Today's flow:

1. `autoCaptureCmd` (cursor up/down) fires `capturePaneCmd(sid)`.
2. Response `paneCapturedMsg{sid, content}` arrives. Handler caches and, if the currently-selected session ID still equals `msg.sid`, schedules `tea.Tick(interval) -> capturePaneTickMsg{sid}`.
3. `capturePaneTickMsg` arrives. If the currently-selected session ID still equals `msg.sid`, fire another `capturePaneCmd`. Otherwise return `m, nil` ‚Äî loop dies.

If the user moves the cursor between firing a capture and receiving its response (easy with the configured 1.5s interval), the response lands with `visible.ID != msg.sid`. The handler returns `m, nil`. No tick is scheduled. The new selection received exactly one capture from `autoCaptureCmd`. The loop is now dead until the next user navigation triggers `autoCaptureCmd` again.

Symptom: preview freezes mid-session.

**Fix:** invert control. Replace the response ‚Üí tick chain with a single ticker driven by `pane_preview_interval`:

```go
type previewTickMsg struct{}

func previewTickCmd(interval time.Duration) tea.Cmd {
    return tea.Tick(interval, func(time.Time) tea.Msg { return previewTickMsg{} })
}
```

In `Update`:

```go
case previewTickMsg:
    next := previewTickCmd(m.ctx.Config.UI.PanePreviewInterval)
    if !m.ctx.Config.UI.ShowPanePreview {
        return m, next
    }
    sess, ok := m.selectedSession()
    if !ok || sess.State.IsFinished() || m.paneCaptureInFlight {
        return m, next
    }
    m.paneCaptureInFlight = true
    return m, tea.Batch(next, capturePaneCmd(m.ctx, sess.ID, m.ctx.Config.UI.PanePreviewLines))
case paneCapturedMsg:
    if m.paneCache == nil {
        m.paneCache = map[string]string{}
    }
    m.paneCache[msg.sid] = msg.content
    m.paneCaptureInFlight = false
    return m, nil
```

`previewTickMsg` always re-arms; can never get stuck. `paneCaptureInFlight` prevents overlapping captures if the agent is slow. `autoCaptureCmd` stays for its current "fire immediately on selection change" purpose.

Initial fire: `Init()` returns `tea.Batch(loadStateCmd, tickStateCmd, previewTickCmd(...))`.

**Bug C ‚Äî no initial capture on first paint.**

`New()` produces an empty `paneCache`. `autoCaptureCmd` only fires from cursor moves. A fresh `cleo` shows "loading‚Ä¶ press v to refresh" until the user navigates.

**Fix:** with the new `previewTickMsg` ticker (Bug B), this is implicit ‚Äî first tick fires within `pane_preview_interval`. No additional code needed.

**Bug D ‚Äî no width truncation on rendered lines.**

`renderPreviewPanel` in `main_pane.go:166-168` does `dimmed.Render(l)` with no width clamp. Lines wider than the inner panel width wrap onto the next visual row, push the bottom border down, and break the layout.

**Fix:** wrap each line through `truncateWidth(line, w-4)` (already used by `renderEventsPanel` and the sidebar):

```go
body[i] = dimmed.Render(truncateWidth(l, w-4))
```

**Bug E ‚Äî empty / whitespace-only pane fallback is misleading.**

If `capture-pane` returns whitespace-only output (agent spawned but never rendered, e.g. `--no-attach`), today the panel reads "loading‚Ä¶ press v to refresh" ‚Äî but `v` won't help (the cache *is* populated, just with whitespace).

**Fix:** distinguish three cases in `renderPreviewPanel`:

```go
pane := m.paneCache[sess.ID]
switch {
case pane == "":
    return m.theme.PanelBox("Terminal Preview", hint,
        []string{faint.Render("loading‚Ä¶")}, w, h)
case strings.TrimSpace(pane) == "":
    return m.theme.PanelBox("Terminal Preview", hint,
        []string{faint.Render("agent hasn't rendered yet ‚Äî press Enter to attach")}, w, h)
default:
    // existing render path
}
```

**Bug F ‚Äî `show_pane_preview = false` ignored by ticker.**

With the new `previewTickMsg` design, the ticker still arms but skips capture if the config flag is off (see Bug B fix). Net cost: a single boolean check every 1.5s. Acceptable.

**File changes.**

```
internal/tmux/tmux.go         (Bug A ‚Äî accept lines parameter)
internal/tmux/tmux_test.go    (test that flag is passed)
internal/tui/poll.go          (drop capturePaneTickMsg, add previewTickMsg)
internal/tui/update.go        (replace capture loop with selection-driven tick)
internal/tui/main_pane.go     (Bugs D, E)
internal/tui/model.go         (add paneCaptureInFlight bool)
internal/tui/run.go           (Init returns batch including previewTickCmd)
internal/tui/tui_test.go      (test capture loop self-recovers across navigation)
```

### 2.2 Esc / Filter / Status Pass

Three coupled UX bugs sharing the Esc and status-line surfaces.

**Esc semantics ‚Äî explicit hierarchy.**

Define in `handle_key.go`:

1. If a popup is open ‚Üí close popup, clear `m.status`, leave filter intact.
2. Else if filter mode active ‚Üí exit filter mode AND clear filter query, restore prior selection by session ID.
3. Else ‚Üí clear `m.status`.

`q` only quits at top level. Inside popups or filter mode, `q` is treated as a regular character.

**Filter persistence across expand/collapse.**

Today, expanding/collapsing a project re-flows the visible row list and may reset filter state. New behavior:

- The filter predicate is re-evaluated against the current row set whenever the visible list changes.
- The filter query string itself is not cleared on structural changes.
- The cursor follows the selected session by ID, not by row index. Helper `m.cursorOnSessionID(string)` recomputes the row index after re-flows.

**Status auto-clear on user input.**

v0.1 clears `m.status` on cursor up/down only. Extend to all user-initiated state changes: filter input, popup open, expand/collapse. Centralize via:

```go
func (m *Model) clearStatusUnless(key string) {
    if m.statusSetByKey != key {
        m.status = ""
    }
    m.statusSetByKey = ""
}
```

Called at the top of the central key handler before dispatch. The `statusSetByKey` field prevents clearing a status that the same key just set.

**File changes.**

```
internal/tui/handle_key.go    (Esc hierarchy, status clear)
internal/tui/filter.go        (filter persistence on re-flow)
internal/tui/model.go         (statusSetByKey field, cursorOnSessionID helper)
internal/tui/tui_test.go      (snapshot test for filter persistence; state tests for Esc)
```

## 3. Reconciler

### 3.1 Synthetic-event clock-reset fix + transition tests

**The bug.** A session that ends in `WaitingForInput` never reaches `Completed`.

Trace:

1. Agent fires `Notification` hook ‚Üí `Running ‚Üí WaitingForInput`. `LastEventAt` updates.
2. User wanders off. `IdleToCompletedTimeout` (default 10m) elapses since `LastEventAt`.
3. Reconciler emits `EvIdleTimeout`. Per `transitions.go`, `WaitingForInput ‚Üí Idle` (the deliberate two-hop design).
4. `state.Apply(EvIdleTimeout)` records the synthetic event ‚Äî and bumps `LastEventAt` to now.
5. Next reconciler cycle: state is `Idle`, but `LastEventAt` was just refreshed in step 4. Idle timer restarts. Goto 5 forever.

`cleo prune` won't clean the session (only operates on finished states). It's stuck.

**Fix.** Synthetic events (`EvIdleTimeout`, `EvDead`) must not bump `LastEventAt`. They're internal bookkeeping.

Concrete change in `internal/state/store.go`: introduce a `synthetic bool` parameter on `Apply`, or add a sibling method `ApplySynthetic`. The `synthetic` path updates `State` and rewrites the session record but does not touch `LastEventAt`. The reconciler's two existing `state.Apply` calls (for `EvIdleTimeout` and `EvDead`) switch to the synthetic path. `EvSessionStart` from the spawning-timeout-advance path is borderline (it's also a reconciler-driven synthesis); keep as a real event so the events log shows the advance.

**Ride-along: unify `Run` and `RunOpts`.**

`internal/cli/ls.go:56` calls `reconcile.Run(...)` which hardcodes `SpawningTimeout = 30s`, ignoring `c.Config.Retention.SpawningTimeout`. Drop `Run` from `internal/reconcile/reconcile.go`. Every caller uses `RunOpts`. One call-site update in `ls.go`.

**Ride-along: visibility for spawning-timeout advance.**

When the reconciler advances `Spawning ‚Üí Running` because `SpawningTimeout` elapsed without a startup hook, today this is silent. Pass a `LastMessage = "advanced from spawning by reconciler (no startup hook seen)"` so the events panel and `cleo events` show what happened.

**Ride-along: reconciler unit tests.**

Add to `internal/reconcile/reconcile_test.go` ‚Äî one test per transition driven by the reconciler:

| Scenario | Expected end state |
|----------|-------------------|
| Session not in tmux ls | `Dead` |
| `Spawning` + tmux alive + `time.Since(StartedAt) > SpawningTimeout` | `Running` (with `LastMessage` set) |
| `Idle` + `time.Since(LastEventAt) > IdleTimeout` | `Completed` |
| `WaitingForInput` + `time.Since(LastEventAt) > IdleTimeout` | `Idle`, `LastEventAt` unchanged |
| Same `WaitingForInput` session, second cycle after another `IdleTimeout` | `Completed` |
| `cleo ls` reconciler call uses configured `SpawningTimeout` | verified by setting config to non-default and asserting timing |

**File changes.**

```
internal/state/store.go             (synthetic Apply path)
internal/state/state_test.go        (test that synthetic events don't bump LastEventAt)
internal/reconcile/reconcile.go     (drop Run; use synthetic path; LastMessage on spawning advance)
internal/reconcile/reconcile_test.go (transition matrix above)
internal/cli/ls.go                  (call RunOpts)
```

### 5.3 Manual verification rituals

Two TUI behaviors that unit tests cannot fully prove. Documented in `CONTRIBUTING.md`:

- **Pane preview correctness.** Open `cleo` against a project with at least one running claude session and one running codex session. Navigate up/down between them rapidly (10+ alternations). The preview must not get stuck on either session's last frame; the visual content for the currently-selected row must update within `pane_preview_interval`. Resize the terminal to roughly half-width ‚Äî panel borders must stay aligned across all rows.
- **Reconciler timing.** Set `idle_to_completed_timeout = 30s`, `spawning_timeout = 10s` in test config. Start a session, immediately fire `Notification`, observe the TUI state column for ~70s. Expected progression: `Spawning ‚Üí Running ‚Üí WaitingForInput ‚Üí Idle ‚Üí Completed`. Without the v0.2 fix, the session stalls at `WaitingForInput` (or oscillates in `Idle`).

### 5.4 PR order and release

Each item is a separate PR. Order:

1. **CI test workflow** (4.1) ‚Äî first; subsequent PRs get feedback. Tiny, low-risk.
2. **Changelog scaffold** (4.2) ‚Äî establishes the per-PR entry pattern.
3. **Reconciler fix** (3.1) ‚Äî most foundational; state correctness affects everything else.
4. **`fallback_reason` traces** (1.3) ‚Äî small, sets up doctor's attribution summary.
5. **`cleo events` command** (1.1).
6. **`cleo doctor` upgrade** (1.2).
7. **Pane preview rewrite** (2.1) ‚Äî biggest visible change; lands after most others to bake.
8. **Esc / filter / status pass** (2.2).
9. **Contributing + issue template** (4.3) ‚Äî last; refers to settled patterns.

**Tagging.** When all 9 PRs merge: rename `[Unreleased]` to `[0.2.0] - <date>`, commit, tag `v0.2.0`, push tag. Goreleaser workflow publishes the GitHub release and updates the homebrew tap. README banner updates from "v0.1 alpha" to "v0.2 ‚Äî pre-1.0, expect breaking changes".

### 5.5 Risks

- **Pane preview rewrite (2.1) is the highest-risk item.** Lots of file changes, multiple bug fixes wrapped in one PR. Mitigation: split if it grows past ~400 lines diff; otherwise ship as one and lean on the manual ritual.
- **Reconciler synthetic-event change (3.1)** has a subtle data-model implication: existing state files written by v0.1 already have `LastEventAt` values that may have been bumped by past synthetic events. v0.2 doesn't backfill or correct them; new behavior takes effect from upgrade onward. Stale "stuck in idle" sessions pre-upgrade can be cleaned with `cleo prune --all` once.
- **`cleo events` archive lookup** depends on `events/archive/` filename layout matching active filenames. Verify: today archives use `<session-id>.jsonl` directly; check that `cleo prune` doesn't rename. (Quick read of `internal/events/archive.go` during implementation.)
