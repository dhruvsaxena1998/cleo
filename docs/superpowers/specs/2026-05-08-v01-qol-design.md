# v0.1 QoL Sweep ‚Äî Design Spec

**Date:** 2026-05-08  
**Scope:** Quality-of-life changes before the v0.1 alpha release. Eight improvements across TUI, CLI, and the focus/sound subsystem, plus one bug fix.

---

## 1. TUI ‚Äî Status Message Auto-Clear

**Problem:** `m.status` is set in error paths (e.g. trying to attach a finished session) but never cleared. It persists in the footer across navigation until another action overwrites it.

**Fix:** Reset `m.status = ""` on every cursor movement (`cursorUp`, `cursorDown`, `toggleExpand`) and when any popup closes (`ConfirmNo`, `RenameCancelled`, `SpawnCancelled`).

No new state required.

---

## 2. TUI ‚Äî `?` Help Overlay

**Problem:** The `?` key hint appears in the footer but pressing `?` does nothing.

**Fix:** Add a `popup_help.go` that renders a two-column table of all keybindings, using the same overlay mechanism as the existing popups. Handle `?` in `handle_key.go` to open `ModePopup` with the help popup. `Esc` or `q` closes it.

Content: group bindings into Navigation, Session Actions, and Global sections. Read the full list from `DefaultKeymap()`.

---

## 3. CLI ‚Äî `doctor` Output Formatting

**Problem:** Output uses `[ok]`/`[warn]` plain text ‚Äî no color, no visual hierarchy.

**Fix:** Replace with `‚úì` (green) and `‚úó` (red) using lipgloss (already a project dependency). Lipgloss handles `NO_COLOR` and non-TTY output automatically. No other changes to `doctor` logic.

Example after:
```
‚úì Claude hooks: 8 hooks installed
‚úó Codex feature flag: missing ~/.codex/config.toml; run cleo init
```

---

## 4. CLI ‚Äî `cleo ls --json`

**Problem:** `cleo ls` is tab-writer formatted, not scriptable.

**Fix:** Add `--json` flag. When set, output a JSON array. Each element:
```json
{
  "project":       "my-project",
  "agent":         "claude",
  "name":          "brave-curie",
  "state":         "running",
  "id":            "cleo-my-project-claude-brave-curie",
  "started_at":    "2026-05-08T10:30:00Z",
  "last_event_at": "2026-05-08T11:02:14Z"
}
```

Projects with no sessions emit one element per project with `agent`, `name`, `state`, `id`, `started_at`, `last_event_at` all `null`.

---

## 5. CLI ‚Äî `cleo ls` Age Column + Sort by Activity

**Problem:** The table has no sense of recency; sessions within a project are in insertion order.

**Fix:**
- Add `AGE` as the rightmost column, showing elapsed time since `LastEventAt` (fallback to `StartedAt`). Format: `5m`, `2h`, `3d`. Show `-` for sessions with no event yet.
- Within each project, sort sessions by `LastEventAt` descending (most recently active first).

---

## 6. CLI ‚Äî `cleo rename <session-id> <new-name>`

**Problem:** Rename is only available in the TUI. There is no CLI equivalent for scripting.

**Fix:** New command `internal/cli/rename.go`:
- Takes exactly two args: `<session-id>` and `<new-name>`
- Slugifies the new name via `ids.Slugify`
- Loads the session from state; errors if not found
- Updates `sess.Name` and calls `c.State.Put(sess)`
- Prints `renamed <session-id>: <old-slug> ‚Üí <new-slug>` (session ID is immutable; only the `Name` field changes)
- Does NOT rename the underlying tmux session (consistent with TUI behavior)

---

## 7. CLI ‚Äî `cleo run` Agent Error Message: Sorted List

**Problem:** `unknown agent "foo" (configured: [claude codex opencode pi])` ‚Äî the list is in random map-iteration order.

**Fix:** Sort `agentKeys(c.Config.Agents)` alphabetically before formatting the error string. One-liner change in `run.go`.

## 8. Bug Fix ‚Äî Focus: Remove `client-focus-out` Hook

**Problem:** Sound plays while the user is physically attached to the claude tmux session.

**Root cause:** `InstallFocusHooks` installs four tmux hooks:
- `client-attached` ‚Üí `cleo focus in #{client_session}`
- `client-focus-in` ‚Üí `cleo focus in #{client_session}`
- `client-focus-out` ‚Üí `cleo focus out #{client_session}`  ‚Üê problematic
- `client-detached` ‚Üí `cleo focus out #{client_session}`

`client-focus-out` fires when the terminal emulator window loses OS focus. This can happen during the alt-screen transition BubbleTea performs when `tea.ExecProcess` starts. Both hooks run as `run-shell -b` (background, asynchronous), creating a write race:
1. TUI calls `Focus.Set(sid, true)` ‚úì
2. `client-focus-out` fires ‚Üí background `cleo focus out <sid>` ‚Üí writes `focused=false`
3. Claude fires `Stop` before `client-attached` has run ‚Üí `IsFocused` returns false ‚Üí sound plays

**Fix:** Remove `client-focus-out` from `InstallFocusHooks`. Keep `client-attached`, `client-focus-in`, and `client-detached`.

**Semantic change:** Sound now plays only when the user has physically detached from the tmux session ‚Äî not when their terminal window momentarily loses OS focus. This is the correct behaviour for the primary use case.
