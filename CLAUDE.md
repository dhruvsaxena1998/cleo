# Claude Instructions

- DO NOT push directly to `main`.
- Always create a feature branch and open a pull request for changes.
- If work is currently on `main`, create a branch before starting the work, committing or pushing.
- ALWAYS run `git branch --show-current` before committing or pushing. A branch you created earlier can be merged and checked out from under you mid-session (e.g. a PR merges and local `main` gets checked out), so never assume you're still on the feature branch — verify first.
- DO NOT add a `Co-Authored-By` / "Generated with Claude Code" footer to commit messages or PR bodies.
