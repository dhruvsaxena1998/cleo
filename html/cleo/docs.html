01 / installInstall Cleo

Cleo runs on macOS and Linux. Windows is not supported in v0.1. Pick whichever install path matches your toolchain; every path installs the same binary.

Prerequisites

  • tmux 3.0 or later on your PATH (runtime).
  • curl for the one-line installer, brew (Homebrew / Linuxbrew), or Go 1.25.5 for go install / building from source.
  • One or more terminal-based agents: claude, codex, opencode, pi, or any other.
  • Sound playback uses afplay on macOS and the first of paplay, aplay, or play on Linux.

Curl installer

# installs the latest GitHub Release to ~/.local/bin
$ curl -fsSL https://github.com/dhruvsaxena1998/cleo/raw/main/scripts/install.sh | sh

The installer supports CLEO_INSTALL_DIR for a custom install directory and CLEO_VERSION for pinned installs.

Homebrew

# 1. add the tap
$ brew tap dhruvsaxena1998/tap

# 2. install cleo
$ brew install cleo

Upgrade

# curl install: re-run the installer
$ curl -fsSL https://github.com/dhruvsaxena1998/cleo/raw/main/scripts/install.sh | sh

# homebrew install: fetch the latest tap and upgrade cleo
$ brew update && brew upgrade cleo

Go install

If you already have Go on your machine, this is the shortest path:

$ go install github.com/dhruvsaxena1998/cleo/cmd/cleo@latest

For local development from a clone of the repo:

$ make build
$ ./bin/cleo --version

Verify the install

$ cleo --version
v0.1.2

If cleo --version prints anything else, see Troubleshooting.

Uninstall

# homebrew install
$ brew uninstall cleo
$ brew untap dhruvsaxena1998/tap

# curl install
$ rm ~/.local/bin/cleo

# go install: remove the binary from $GOBIN / $GOPATH/bin
$ rm "$(which cleo)"
i
Uninstalling Cleo does not remove your sessions from tmux. Run tmux kill-server if you want to wipe them too. Cleo never owns the underlying tmux server. To also clear hook entries from your agents, run cleo hooks cleanup before removing the binary.

02 / quick startFrom zero to your first session

Four commands to get from "installed" to "watching an agent work".

# 1. install hooks for claude code & codex (once per machine)
$ cleo init

# 2. register a project
$ cd ~/Dev/myapp
$ cleo add

# 3. spawn a claude session inside it
$ cleo run claude --name fix-auth-bug

# 4. open the dashboard whenever you want it
$ cleo

Cleo creates a tmux session named cleo-myapp-claude-fix-auth-bug, launches claude inside the project's working directory, and attaches the hook listener. Detach with the configured tmux key (default Ctrl-b d); the session keeps running.

i
If you don't pass --name, Cleo assigns a Docker-style label like brave-curie. Cleo also auto-registers a project on cleo run if you skipped cleo add; pass --yes to skip the confirmation prompt.

03 / commandsThe CLI surface

Cleo's CLI is small on purpose. Most of the work happens inside the dashboard's keybindings.

Setup & diagnostics

cleo Open the TUI dashboard. The default action with no arguments.
cleo hooks init [--yes] [--force] Install Cleo hook entries into ~/.claude/settings.json and ~/.codex/hooks.json, and extract bundled sound assets.
cleo doctor Verify hook entries, the Codex feature flag, and recent hook-trace activity.
cleo hooks cleanup [--yes] Remove Cleo hook entries from agent config files. Same as cleo uninstall.
cleo --version Print the installed version.

Projects

cleo add [path] Register a project. Defaults to the current directory. Project IDs are slugified from the directory name.
cleo rm <project> Unregister a project from Cleo. Running tmux sessions keep running; your code is untouched.
cleo ls List registered projects and their known sessions: project ID, agent, name, state, full session ID.

Sessions

cleo run <agent> [opts] Spawn an agent in tmux. Options: --name, --cwd, --yes, --no-attach. Session ID format: cleo-<project>-<agent>-<name>. Without --name, Cleo assigns a Docker-style label like brave-curie.
cleo attach <session-id> Attach to an existing tmux session. Detach with the configured tmux detach key.
cleo rename <session-id> <name> Rename the Cleo-side label only. The underlying tmux ID is preserved, so existing cleo attach commands and hook attribution keep working unchanged. Inside the TUI, press r.
cleo kill <session-id> [--yes] Kill a running tmux session and remove it from Cleo state.
cleo prune [project] [opts] Remove finished (completed / error / dead) sessions and archive their event logs. Options: --keep N, --all, --dry-run, --yes. 
# a few common shapes of `cleo run`
$ cleo run claude
$ cleo run codex --name refactor-auth
$ cleo run claude --cwd ~/Dev/myapp --name fix-tests --no-attach
$ cleo run codex --yes

Inside the TUI dashboard

When you run cleo with no arguments, the same operations are one keystroke away. The footer always shows the action list for the selected row, so the live UI is the safest source of truth.

↵  / enterAttach to the selected session.

04 / hooks & stateHow Cleo knows what's happening

Cleo treats sessions as objects with state. It learns that state by listening to hook events emitted by supported agents. Claude Code, Codex, Pi, and OpenCode have hook integrations installed by cleo hooks init. Other terminal agents can still be spawned and managed through tmux, with less detailed lifecycle tracking.

Run cleo hooks init once per machine. Cleo writes hook entries to the agent config files, then writes CLEO_SESSION_ID into each tmux session's environment so hook payloads can be attributed to the right session.

Claude Code

Hook entries land in ~/.claude/settings.json. Cleo registers itself for these events:

SessionStart
UserPromptSubmit
PreToolUse
PostToolUse
Notification
Stop
SessionEnd
SubagentStop

Codex

Hook entries land in ~/.codex/hooks.json, plus cleo hooks init ensures [features].hooks = true exists in ~/.codex/config.toml. Cleo registers itself for:

SessionStart
UserPromptSubmit
PreToolUse
PostToolUse
PermissionRequest
Stop
!
After installing Codex hooks, restart any open Codex sessions and run /hooks inside Codex to approve the Cleo entries if they appear under review. Codex keeps approval state internally; cleo doctor can verify the files but cannot prove approval.

Pi & OpenCode

Pi uses a TypeScript extension at ~/.pi/agent/extensions/cleo.ts. OpenCode uses a TypeScript plugin at ~/.config/opencode/plugins/cleo.ts. Both are installed by cleo hooks init and checked by cleo doctor.

Pi:       session_start · input · tool_call · tool_result · agent_end · session_shutdown
OpenCode: session.created · tool.execute.before · tool.execute.after · permission.asked · session.idle · session.deleted · session.error

Session states

spawningCleo created state and is starting tmux/agent.
runningThe agent is active or has recently resumed work.
waiting_for_inputThe agent requested input, approval, or attention.
idleThe agent stopped its current turn but the session is still available.
completedThe session ended cleanly or aged out from idle.
errorCleo recorded an error state.
deadThe underlying tmux session is gone.

Where events go. Each hook event is appended to ~/.config/cleo/events/<session-id>.jsonl. When you cleo prune a session, its log is moved to ~/.config/cleo/events/archive/ rather than deleted.

05 / configurationState, sounds, and where things live

Cleo stores everything in your XDG config dir. No daemon, no service, no remote state. The config file is created on first run with sensible defaults; partial configs are supported, so you can keep config.toml minimal and only override what you actually want to change.

Files & locations

# default; respects $XDG_CONFIG_HOME if set
~/.config/cleo/
├─ config.toml           # user config (created on first run)
├─ projects.json         # registered projects
├─ state.json            # current known sessions
├─ state.json.lock       # state file lock
├─ focus.json            # tmux focus state for sound suppression
├─ events/               # <session-id>.jsonl per session
│   └─ archive/           # pruned session logs
├─ sounds/               # bundled .wav files (override here)
├─ hook-errors.log       # hook handler errors
└─ hook-trace.log        # attribution trace used by `cleo doctor`

Hook entries themselves live with the agent: ~/.claude/settings.json for Claude Code, ~/.codex/hooks.json + ~/.codex/config.toml for Codex, ~/.pi/agent/extensions/cleo.ts for Pi, and ~/.config/opencode/plugins/cleo.ts for OpenCode.

Cleo reads config.toml on startup. After editing it, quit the dashboard (q) and relaunch cleo to pick up changes. Hook handlers re-read the config on every invocation, so sound and event toggles take effect on the next hook event without restarting anything. Durations use Go duration strings: "500ms", "1.5s", "30s", "10m", "1h".

Top-level keys

default_agent = "claude"
default_agentDefault agent name for flows that need one but didn't get one explicitly.

[tmux]

[tmux]
detach_key = "C-b d"
detach_keyTmux detach key Cleo tries to bind for spawned sessions. Default C-b d.

[sound]

[sound]
enabled = true
volume  = 0.7

[sound.events.session_start]
file    = "start.wav"
enabled = true

[sound.events.needs_input]
file    = "attention.wav"
enabled = true

[sound.events.session_idle]
file    = "done.wav"
enabled = true

[sound.events.session_completed]
file    = "done.wav"
enabled = true

[sound.events.session_error]
file    = "error.wav"
enabled = true
enabledMaster switch for sound playback on hook-triggered transitions.
volume0.0–1.0. Applied on macOS via afplay -v; Linux players (paplay/aplay/play) ignore it; use the system mixer there.
[sound.events.<event>]Configures one event's file and enabled fields. Relative paths resolve under ~/.config/cleo/sounds/; absolute paths are used as-is.

Custom sounds: drop your own .wav into ~/.config/cleo/sounds/ and reference it by basename, or use an absolute path. cleo hooks init extracts the bundled defaults start.wav, attention.wav, done.wav, and error.wav.

Focus-aware suppression. Cleo enables tmux focus-events and installs client focus hooks when you start or attach to a session. While you're focused on that exact tmux session, its sounds are suppressed to avoid duplicate alerts; switching to another app (VS Code, Chrome) emits focus-out and Cleo resumes notifications. This is best-effort; when focus state can't be determined, Cleo prefers to play sounds rather than risk hiding an alert.

Configuration recipes

The defaults are reasonable, so most users only need a few overrides. A few common patterns:

Silent mode: disable all sound playback without removing event mappings:

[sound]
enabled = false

Only attention-worthy sounds: mute routine start / idle / completion, keep the ones that signal you should look at the terminal:

[sound.events.session_start]
enabled = false

[sound.events.session_idle]
enabled = false

[sound.events.session_completed]
enabled = false

Lower default volume (macOS): Linux uses the system mixer:

[sound]
enabled = true
volume  = 0.3

Custom sound files:

[sound.events.needs_input]
file = "my-attention.wav"

[sound.events.session_error]
file = "/Users/me/sounds/error-loud.wav"

Switch theme:

[ui]
theme = "synthwave"

Add a new agent: example for aider:

[agents.aider]
command = "aider"
label   = "ai"
color   = "#9333EA"

Then cleo run aider --name refactor-payments.

Override an existing agent's command: flags and arguments are preserved verbatim:

[agents.claude]
command = "claude --dangerously-skip-permissions"
label   = "cl"
color   = "#CC785C"

Larger preview pane / more breathing room:

[ui]
sidebar_width         = 40
event_log_lines       = 500

[ui.pane_preview]
lines    = 60
interval = "0.75s"

Tighter pruning and timeouts: auto-promote idle sessions sooner and keep fewer finished sessions per project:

[timeouts]
idle_to_completed_timeout = "2m"
spawning_timeout          = "15s"

[pruning]
hint_threshold = 3
keep_default   = 2

Change the default agent:

default_agent = "codex"

Environment variables

Cleo reads only a small set of environment variables.

XDG_CONFIG_HOMEOverrides the config root. When set, Cleo uses $XDG_CONFIG_HOME/cleo/ instead of ~/.config/cleo/.
CLEO_SESSION_IDSet automatically by cleo run inside the spawned tmux session. Hook handlers read it to attribute events to the correct Cleo session; you normally don't set it yourself.
TMUXStandard tmux variable. Cleo checks it to detect that you're already inside tmux when running cleo run, so it can switch-client rather than nest a new server.
i
If you delete or corrupt config.toml, the next cleo run rewrites it from defaults. Your projects and session state live in separate files (projects.json, state.json) and are unaffected.

06 / troubleshootingCommon issues

cleo hooks init reports a hook conflict

The target agent config already has a different entry for the same event. Review the file by hand, or rerun with --force to overwrite Cleo-managed events:

$ cleo hooks init --force

Codex hooks are installed but nothing updates

Run cleo doctor, then open Codex and run /hooks; approve the Cleo hook names if they're under review. Restart any Codex sessions opened before cleo hooks init; they may not have loaded the updated ~/.codex/config.toml.

Sessions stay running

For custom agents without a Cleo hook integration, this can be expected. For supported agents, run cleo doctor and check:

$ tail -n 50 ~/.config/cleo/hook-trace.log
$ tail -n 50 ~/.config/cleo/hook-errors.log

A session is dead

The tmux session no longer exists: killed externally, exited, or removed by another tool. Clean it up with:

$ cleo prune

No sound plays

  • Check cleo hooks init ran and ~/.config/cleo/sounds/ contains the WAV files.
  • On macOS, confirm which afplay resolves.
  • On Linux, Cleo uses the first available of paplay, aplay, or play.
  • Disable entirely with [sound] enabled = false in config.toml.

Cleo binary moved after hooks were installed

Hook entries store the absolute path to the cleo executable. Re-run:

$ cleo hooks init

so hook files point at the current binary path.

Project was registered with the wrong path

Remove and re-add it:

$ cleo rm old-project-id
$ cleo add /correct/path

"tmux: command not found"

Cleo doesn't ship tmux; it talks to your installed copy. Install tmux 3.0+ via Homebrew:

$ brew install tmux

Still stuck?

Open an issue at github.com/dhruvsaxena1998/cleo/issues with the output of cleo --version, tmux -V, cleo doctor, and your platform.
