how it works

Three steps, one local-first loop. No daemon. No service process. No babysitting.

You spawn the agents. Cleo stalks them. They've got nine lives in tmux.

Cleo creates a tmux session with a stable ID like cleo-invoice-api-claude-fix-auth and launches the agent inside it. No --name? You get a Docker-style label like brave-curie instead.

$ cleo add ~/Dev/invoice-api
✓ registered     invoice-api

$ cleo run claude --name fix-auth
✓ tmux session   cleo-invoice-api…
✓ hooks attached
↳ attaching…

Claude Code and Codex emit hook events on start, prompt, tool use, and stop. Cleo logs each one to events/<id>.jsonl and chimes for the moments worth interrupting for. Mute with alt+m when she gets too needy.

17:02:14  ⏵ SessionStart
17:09:05  ⌥ PreToolUse · Edit
17:18:47  ⌫ Notification · awaiting input
17:34:02  ✓ Stop · turn complete
17:34:02  ♪ chime → done.wav

Sessions live in tmux, so closing Cleo doesn't end them. Patient as a cat. The agents keep working, the events keep logging. Reopen the dashboard whenever, attach when you're ready, prune when you're done.

$ cleo
4 sessions · 2 live · 1 waiting

▸ invoice-api           1 session

what's in the basket

Six things Cleo does so you don't have to remember tmux or your last six terminal tabs.

Built around a simple idea: agent sessions deserve nine lives.

01

Named tmux sessions

Spawn agents into stable, readable session IDs like cleo-myapp-claude-fix-auth. If you don't pass --name, you get a Docker-style label like brave-curie.

02

One dashboard, every session

All registered projects and their sessions in a single TUI. Filter with /, attach with ↵, view with v, send input with m, kill with K.

03

Hook-aware state

Reads Claude Code, Codex, OpenCode, and Pi hooks to track spawning · running · waiting_for_input · idle · completed · error · dead. Custom agents still run as managed tmux sessions.

04

Audible transitions

Plays a local WAV for the moments worth interrupting for: session start, needs input, idle, completed, error. Mute with alt+m; auto-suppresses while you're focused on the session.

05

Per-session event log

Every hook event is timestamped and written to ~/.config/cleo/events/<id>.jsonl. cleo prune archives finished sessions; the logs survive.

06

Local-first, no daemon

State lives in ~/.config/cleo/ (or $XDG_CONFIG_HOME/cleo/). No background service, no cloud, no telemetry. Just a TUI you open when you want it.

install

One line. One install.

Install the latest GitHub Release on macOS or Linux with curl. Homebrew stays supported if you prefer a package manager; either way, you'll need tmux available on your PATH.

read the docs  → view source
1 · curl install
# installs the latest GitHub Release to ~/.local/bin
$ curl -fsSL https://github.com/dhruvsaxena1998/cleo/raw/main/scripts/install.sh | sh
2 · verify
$ cleo --version
v0.2.2
homebrew option
# package-manager install
$ brew tap dhruvsaxena1998/tap
$ brew install cleo
3 · install hooks for supported agents
# writes hook entries for claude, codex, pi, and opencode
$ cleo hooks init
✓ claude code hooks
✓ codex hooks
✓ pi extension
✓ opencode plugin
✓ sound assets extracted
4 · register a project & spawn an agent
$ cd ~/Dev/myapp
$ cleo add
$ cleo run claude --name fix-auth-bug
$ cleo                 # open the dashboard
cleanup (when you're done)
# curl install
$ rm ~/.local/bin/cleo

# homebrew install
$ brew uninstall cleo
$ brew untap dhruvsaxena1998/tap
